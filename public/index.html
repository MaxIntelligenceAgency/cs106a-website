02 · Motivation

Convoying is mature in cars. Under-exploited for drones.

In domains where full autonomy is infeasible — inspection, search-and-rescue, warehouse logistics — convoying lets one operator command many robots with high precision and lower risk than running each independently. The pattern is well-understood for ground vehicles. It is barely touched for UAVs.

We prototype that idea on compute-constrained UAV hardware: a $100 self-funded Tello drone with a 720p / 30 fps camera, a base-station laptop running ROS 2 Humble, and a Wi-Fi link doing all the heavy lifting.

03 · System Architecture

Two perception pipelines. One shared ROS bus.

Used ROS 2 Humble for communication between the base station and the drone. Both modes publish the same Twist command. Only the perception source changes; safety actions stay unified through /tello_action.

ArUco perception

Detect the printed ArUco marker and turn it into a 6-DoF pose the controller can servo on.
Reading aruco_perception.py · 1.2 s

Run cv2.aruco.detectMarkers on every camera frame. When ID 0 is visible, run solvePnP with the printed tag size and camera intrinsics — that gives a direct metric translation and rotation.

The yaw is extracted from the rotation matrix and packed into the unused orientation.z field so messaging stays cheap.

# aruco_perception.py · corners + tag size → solvePnP → 6-DoF pose
def on_frame(self, frame):
    corners, ids, _ = cv2.aruco.detectMarkers(frame, self.dict)
    if ids is None:
        return                            # tag not visible this frame

    rvec, tvec = cv2.solvePnP(corners, self.tag_size,
                                  self.K, self.dist)

    pose = Pose()
    pose.position.x, pose.position.y, pose.position.z = tvec
    R, _ = cv2.Rodrigues(rvec)
    pose.orientation.z = math.atan2(
        -R[2,0],
        math.sqrt(R[2,1]**2 + R[2,2]**2),
    )
    self.pose_pub.publish(pose)            # → /aruco/pose_3d
# face_perception.py · YOLOv8 box + ReID → center / area pose
def on_frame(self, frame):
    boxes = self.yolo(frame).boxes
    face  = self.select_target(boxes, self.reid)
    if face is None:
        return self.haar_fallback(frame)   # transient YOLO loss

    H, W = frame.shape[:2]
    pose = Pose()
    pose.position.x = (face.cx - W/2) / (W/2)   # horizontal err
    pose.position.y = (face.cy - H/2) / (H/2)   # vertical err
    pose.position.z = face.area                  # depth proxy

    self.pose_pub.publish(pose)            # → /face/pose
# aruco_controller.py · metric pose servo
# e = [−x, z − d_target, −y, −yaw]
def step(self, pose):
    x, y, z   = pose.position.x, pose.position.y, pose.position.z
    yaw_tag   = pose.orientation.z

    vx  = Kp_fwd  * (z - target_dist) + Kd_fwd  * LPF(self.de_fwd)
    vy  = Kp_side * (-x)              + Kd_side * LPF(self.de_x)
    vz  = Kp_z    * (-y)              + Kd_z    * LPF(self.de_y)
    yaw = Kp_yaw  * (-yaw_tag)        + Kd_yaw  * LPF(self.de_yaw)

    # EMA smoothing α = 0.4 · RC clamps 12 / 18 / 12 / 25
    self.publish_twist(clamp(vx, vy, vz, yaw))
# face_controller.py · image-based servo
# e = [x_center, y_center, A_target − A_face]
def step(self, pose):
    err_x    = pose.position.x                  # normalized [-1, 1]
    err_y    = pose.position.y
    err_area = target_area - pose.position.z    # depth proxy

    yaw = Kp_yaw * err_x    + Kd_yaw * LPF(self.de_x)
    vz  = Kp_z   * (-err_y) + Kd_z   * LPF(self.de_y)
    vx  = Kp_fwd * err_area + Kd_fwd * LPF(self.de_area)

    # Holds head near image center · ~1.5 m social distance
    self.publish_twist(clamp(vx, 0.0, vz, yaw))
# state_machine.py · 5-state safety machine
def step(self, pose, dt):
    if   self.state == IDLE:
        if pose and self.tag_seen_for(2.0):
            self.state = ARM
            self.takeoff()                  # → /tello_action

    elif self.state == FOLLOW:
        if pose is None:
            self.lost_t += dt
            if self.lost_t > 0.5:
                self.state = SEARCH
        else:
            self.lost_t = 0.0
            self.controller.step(pose)        # PD + LPF

    elif self.state == SEARCH:
        self.yaw_scan()
        if   pose:               self.state = FOLLOW
        elif self.lost_t > 5.0: self.state = LAND

    elif self.state == LAND:
        self.soft_land()                    # → /tello_action
05 · Operation

Five-state safety machine.

The leader takes off and is teleoperated. The follower confirms the tag, takes off, locks at safe distance, copies the leader's velocity, recovers if the tag is lost briefly, and lands cleanly on long loss.

01 · IDLE
Idle
Disarmed. Waiting for tag & takeoff.
02 · ARM
Arm
Stable tag for ~2 s. Auto-takeoff.
03 · FOLLOW
Follow
Track the tag at safe distance.
04 · SEARCH
Search
Tag lost < 2 s. Hover, yaw scan, retry.
05 · LAND
Land
Tag lost > 5 s. Soft-land. Disarm.

08 · Failures & Mitigations

Six issues we hit — and the fixes that ship.

▲ 01 Wi-Fi drops & network in VMs Controlled timeout for Wi-Fi instability; switched from Docker (system-agnostic) to WSL on Windows so the Tello UDP socket maps directly through the host.
▲ 02 ArUco pose flicker EMA smoothing filter on the translation component of the pose.
▲ 03 Sudden tracking errors PD control with a low-pass filter cleans up noisy pose estimates before they reach the actuator.
▲ 04 Half-second occlusions Search state machine: hover, yaw-scan, retry, fall through to safe-landing if the scan exhausts.
▲ 05 Multi-node Tello IO latency Single owner pattern — one tello_direct_io_node wraps the DJI Tello packages and handles everything end-to-end.
▲ 06 Inconsistent video stream Smoothed video quality through Tekko frame handling, and moved detection in-process to remove the ROS-image round trip.
09 · Future Plans

Four directions we want to take this.

→ 01 Voice-selectable tracking A multimodal operator command to switch between tracking modes — ArUco, face, or full-person. Voice frees the operator's hands during a demo.
→ 02 Multi-follower formation Use shared TF frames so multiple drones maintain formation offsets from the leader instead of all targeting the same point.
→ 03 Mental-mapping replay The follower remembers another drone's path and replays it later with obstacle awareness.
→ 04 Onboard inference Move perception to a Jetson companion to cut Wi-Fi latency and drop the base-station dependency.
10 · Real-hardware bridges

Two utility scripts that target the physical Tello.

≋

tello-station.sh

One-shot helper that switches the Tello from its default AP mode into station mode, joining a target Wi-Fi (e.g. a phone hotspot). Verifies the Mac is on the TELLO-XXXXXX hotspot, pings the drone, sends command / battery? / ap <ssid> <pass> over UDP 8889. Refuses to run below 25 % battery.

▦

tt_show_aruco.py

Renders a real ArUco marker directly onto the RoboMaster TT's top 8×8 LED matrix. A TT can literally carry its own ArUco for another drone to track — physical equivalent of the moving marker in the simulation demo.

11 · Quickstart

How to run it.

Requires Ubuntu 22.04 (or WSL 2 on Windows) + ROS 2 Humble + Gazebo Classic 11. Every fresh terminal must source the workspace.

$ bash
# ① Build the workspace
cd ~/ros2_ws/106a_final_project/drone_simulation
source /opt/ros/humble/setup.bash
colcon build
# ② Sim path · launch Gazebo + simulated Tello
source install/setup.bash
ros2 launch tello_gazebo simple_launch.py

# second terminal · perception + control
ros2 launch my_drone_vision vision_control.launch.py

# third terminal · marker trajectory
python3 marker_move.py
# ③ Real path · ArUco follow on a Tello
# put Tello onto the local Wi-Fi (one-shot)
./tello-station.sh

# launch the I/O + controller node pair
ros2 launch my_drone_vision tello_aruco.launch.py
# ④ Real path · face follow
ros2 launch my_drone_vision tello_face.launch.py
# YOLOv8 weights are downloaded the first time

pkill -f "gazebo|gzserver|gzclient"

↕

Marker won't move on z

Gazebo is using an old model with planar_move. Restart Gazebo and confirm the model comes from this repo.

×

No marker / face detected

Check the camera topic has data, and that the perception node was started after sourcing install/setup.bash.
